"Give me users"

Database lookup

JSON data back

Enter Fetch API

The Fetch API

Built into every modern browser. No libraries needed. Makes HTTP requests from JavaScript.

// The most basic fetch call
fetch('https://api.example.com/users')
Built-in
No npm install, no CDN, always available
Promise-based
Works perfectly with async/await
Replaced XMLHttpRequest
The old way was ugly. Fetch is clean.

07

The Response Object

const response = await fetch('/api/users');

// Response properties:
response.ok      // true if status 200-299
response.status    // 200, 404, 500, etc.
response.statusText // "OK", "Not Found", etc.
response.headers   // ResponseHeaders object

// Body parsing methods (each returns a Promise!):
response.json()     // → Parse as JSON
response.text()     // → Parse as string
response.blob()     // → Parse as Blob (files)
response.json() returns a Promise — you must await it separately!
08

Parsing JSON

// Step 1: Fetch
const response = await fetch('https://jsonplaceholder.typicode.com/users');

// Step 2: Parse (DON'T FORGET THIS!)
const users = await response.json();

// Step 3: Use the data
console.log(users[0].name); // "Leanne Graham"
console.log(users[0].email); // "Sincere@april.biz"
Without .json(), response is just a stream — not usable data!
What JSON looks like:
[{"id": 1, "name": "Leanne", "email": "..."}, ...]

11
The Solution

Async / Await

Syntactic sugar over Promises. Makes async code look synchronous.

// Same logic, but readable!
async function getUsers() {
  const response = await fetch('/api/users');
  const users = await response.json();
  console.log(users);
}
async keyword marks a function as asynchronous
await pauses execution until the Promise resolves
Code reads top to bottom — like normal code
12

Async Function Rules

Rule 1 async keyword
// Function declaration
async function getData() { ... }

// Arrow function
const getData = async () => { ... };

// ⚠️ await only works INSIDE async functions
Rule 2 await keyword
const data = await fetch(url); // pauses here
// doesn't run next line until fetch completes
console.log(data); // guaranteed to have data
Rule 3 Returns a Promise
async function foo() { return 42; }
foo().then(v => console.log(v)); // 42
13

What Await Actually Does

execution timeline
→ Line 1: fetch() starts (network request sent)
⏸ Line 1: await pauses this function
... other code can run ...
... UI stays responsive ...
▶ Line 1: response arrives, function resumes
→ Line 2: response.json() starts
⏸ Line 2: await pauses again
▶ Line 2: parsed data available
→ Line 3: console.log(users) ✓
await ≠ block. The whole page doesn't freeze. Only that function pauses.

.then() vs async/await

.then() Chaining
fetch(url)
 .then(r => r.json())
 .then(data => {
  console.log(data);
 })
 .catch(err => {
  console.error(err);
 });
async/await ✨
async function load() {
  try {
    const r = await fetch(url);
    const d = await r.json();
    console.log(d);
  } catch(err) {
    console.error(err);
  }
}
Same result. async/await is just cleaner syntax.
15

Error Handling: try/catch

async function getUsers() {
  try {
    const response = await fetch('/api/users');

    // Check for HTTP errors!
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}`);
    }

    const users = await response.json();
    return users;

  } catch (error) {
    console.error('Failed:', error.message);
    return []; // graceful fallback
  }
}
fetch doesn't throw on 404/500! It only throws on network failure. Always check response.ok.

Rendering API Data to DOM

async function displayUsers() {
  const res = await fetch('/api/users');
  const users = await res.json();

  const container = document.querySelector('#user-list');

  users.forEach(user => {
    const div = document.createElement('div');
    div.innerHTML = `
      <h3>${user.name}</h3>
      <p>${user.email}</p>
    `;
    container.appendChild(div);
  });
}

POST Request — Sending Data

async function createPost() {
  const newPost = {
    title: 'My First Post',
    body: 'This is the content...',
    userId: 1
  };

  const res = await fetch('https://jsonplaceholder.typicode.com/posts', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(newPost)
  });

  const data = await res.json();
  console.log(data); // { id: 101, title: "..." }
}

POST Options Breakdown

method
Set to 'POST' to send data. Default is 'GET'.
headers
Tells the server what format the data is in:
{ 'Content-Type': 'application/json' }
// "Hey server, this body is JSON"
body
The actual data. Must be a string:
JSON.stringify({ name: 'John' })
// → '{"name":"John"}'

// ❌ WRONG: body: { name: 'John' }

Error States

Network Error
No internet, server down, CORS blocked. fetch() throws.
catch(err) { // "Failed to fetch" }
HTTP Error (4xx, 5xx)
404 Not Found, 401 Unauthorized, 500 Server Error. fetch() does NOT throw!
if (!res.ok) throw new Error(res.status);
Parse Error
Server returns invalid JSON. .json() throws.
const data = await res.json(); // throws

Complete Async Pattern

async function loadDashboard() {
  const list = document.querySelector('#list');
  const loader = document.querySelector('#loader');

  // 1. Show loading
  list.innerHTML = '';
  loader.classList.remove('hidden');

  try {
    // 2. Fetch
    const res = await fetch('https://jsonplaceholder.typicode.com/users');
    if (!res.ok) throw new Error(`HTTP ${res.status}`);

    // 3. Parse
    const users = await res.json();

    // 4. Render
    users.forEach(u => {
      list.innerHTML += `<div>${u.name} - ${u.email}</div>`;
    });

  } catch(err) {
    // 5. Error state
    list.innerHTML = `<p style="color:red">${err.message}</p>`;
  } finally {
    // 6. Hide loading (always!)
    loader.classList.add('hidden');
  }
}
